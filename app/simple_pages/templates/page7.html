OOP Demonstration in Calculator App

My Calculator app is designed on the principles of Object oriented principles.With the help of calculator app ,I am going to show how the four major concepts of Encapsulation,Inheritance,Abstraction and Polymorphism were implemented

Encapsulation

Encapsulation is the idea of wrapping data and the methods that work on data within one unit. This puts restrictions on accessing variables and methods directly and can prevent the accidental modification of data.

In our calculator app , this principle is demonstrated by calculator class.

Inside the static methods subtract and multiply ,If you observe the calculation variable holds the object of class subtract and multiply.It encapsulates the functionality and the list of data that the operation needs to be performed to get the result

Inheritance

Inheritance is the capability of one class to derive or inherit the properties from another class. It also gives child class the abiltiy to override the functionality of the inherited class from the parent class

In our calculator app , this principle is demonstrated by calculation class.

If you observe the parent calculation class has two methods to create a tuple list and convert the values to float

when our child class Addition extends the parent class ,the methods of parent class are inherited in Addition class. It will create a tuple list of your inputs and convert the values to float without you specifying them. This methods can be reused in any number of child classes extended from Calculation parent class.

Abstraction

An abstract class can be considered as a blueprint for other classes. A class which contains one or more abstract methods is called an abstract class. An abstract method is a method that has a declaration but does not have an implementation.Abstraction is used to hide the internal functionality of the function from the users. The users only interact with the basic implementation of the function, but inner working is hidden.

In our calculator app , this principle is demonstrated as follows:

Inside the main calculator class ,we call the multiplication class from calculations class

Inside the multiplication class ,we call the get result method

Inside the get result method, the multiply method from operations is called

Finally only output of get result method is retrieved from the main calculator class
Polymorphism

The word polymorphism means having many forms. In programming, polymorphism means the same function name (but different signatures) being used for different types.

In our calculator app , this principle is demonstrated as follows:

Inside the main calculator class ,we call the get_result() method which can be called on any operation.
